# ✅ ПРОВЕРКА АВТОЗАПУСКА PM2

## 🔍 КАК ПРОВЕРИТЬ:

### Шаг 1: Проверить, что PM2 сохранен
```bash
pm2 save
```

### Шаг 2: Проверить, настроен ли автозапуск
```bash
pm2 startup
```

Если автозапуск НЕ настроен, команда покажет что-то вроде:
```
[PM2] Init System found: systemd
[PM2] To setup the Startup Script, copy/paste the following command:
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root
```

### Шаг 3: Выполнить команду (если показана)
Скопируйте и выполните команду, которую показал PM2.

### Шаг 4: Проверить systemd сервис
```bash
systemctl status pm2-root
# или
systemctl is-enabled pm2-root
```

Должно быть: `enabled`

---

## 🚀 НАСТРОЙКА АВТОЗАПУСКА (если не настроен):

### Вариант 1: Через PM2 (рекомендуется)

```bash
# 1. Сохранить текущие процессы
pm2 save

# 2. Настроить автозапуск
pm2 startup

# 3. Выполнить команду, которую покажет PM2
# (обычно что-то вроде):
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root

# 4. Проверить
systemctl status pm2-root
```

### Вариант 2: Вручную через systemd

Создать файл `/etc/systemd/system/pm2-root.service`:

```ini
[Unit]
Description=PM2 process manager
Documentation=https://pm2.keymetrics.io/
After=network.target

[Service]
Type=forking
User=root
LimitNOFILE=infinity
LimitNPROC=infinity
LimitCORE=infinity
Environment=PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin
Environment=PM2_HOME=/root/.pm2
PIDFile=/root/.pm2/pm2.pid
Restart=on-failure

ExecStart=/usr/local/lib/node_modules/pm2/bin/pm2 resurrect
ExecReload=/usr/local/lib/node_modules/pm2/bin/pm2 reload all
ExecStop=/usr/local/lib/node_modules/pm2/bin/pm2 kill

[Install]
WantedBy=multi-user.target
```

Затем:
```bash
sudo systemctl daemon-reload
sudo systemctl enable pm2-root
sudo systemctl start pm2-root
```

## ✅ ПРОВЕРКА:

### Тест 1: Проверить статус
```bash
pm2 status
systemctl status pm2-root
```

### Тест 2: Симуляция перезагрузки
```bash
# Остановить PM2
pm2 kill

# Запустить через systemd
sudo systemctl start pm2-root

# Проверить, что процессы восстановились
pm2 status
```

### Тест 3: Настоящая перезагрузка (опционально)
```bash
# Перезагрузить сервер
sudo reboot

# После перезагрузки проверить
pm2 status
# Должен показать все процессы онлайн
```

## ❌ ЕСЛИ НЕ РАБОТАЕТ:

### Проблема: PM2 не запускается после перезагрузки

**Решение:**
1. Проверить, что `pm2 save` был выполнен
2. Проверить, что systemd сервис создан: `systemctl status pm2-root`
3. Проверить логи: `journalctl -u pm2-root -n 50`

### Проблема: Процессы не восстанавливаются

**Решение:**
```bash
# Пересохранить
pm2 save

# Проверить dump файл
ls -la /root/.pm2/dump.pm2

# Пересоздать автозапуск
pm2 unstartup
pm2 startup
# Выполнить команду
```
