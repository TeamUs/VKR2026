# ✅ ПРАВИЛЬНАЯ КОНФИГУРАЦИЯ PRODUCTION

## 🎯 КАК ЭТО РАБОТАЕТ:

### Development (локально):
```
Frontend: npm run dev  → Vite dev server (порт 3001) - процесс
Backend:  npm start    → Node.js процесс (порт 3000)
```

### Production (на сервере):
```
Frontend: npm run build → статические файлы в dist/ (НЕ процесс!)
Backend:  PM2          → Node.js процесс (порт 3000) - процесс
Nginx:                 → раздает frontend файлы + проксирует API
```

---

## ✅ ПРАВИЛЬНАЯ НАСТРОЙКА:

### 1. PM2 управляет ТОЛЬКО backend:

```javascript
// ecosystem.config.js
{
  name: 'poizonic-backend',
  script: './backend/server.js',  // Только backend!
}
```

### 2. Frontend собирается отдельно (один раз):

```bash
cd frontend
npm run build  # Создает статические файлы в dist/
```

### 3. Nginx раздает frontend и проксирует backend:

- Статические файлы из `dist/` отдаются как обычный сайт
- API запросы `/api/*` проксируются на `http://localhost:3000`

---

## 🔄 ЧТО ДЕЛАЕТСЯ ПРИ ДЕПЛОЕ:

### Вариант 1: Ручной деплой

```bash
# 1. Обновить код
git pull origin main

# 2. Собрать frontend
cd frontend
npm run build

# 3. Перезапустить backend (если нужен)
pm2 restart poizonic-backend
```

### Вариант 2: Автоматический деплой (скрипт)

Можно создать скрипт `deploy.sh`, который:
1. Обновляет код
2. Собирает frontend
3. Перезапускает backend

Но PM2 управляет только backend процессом!

---

## ❌ ЧТО НЕ НУЖНО:

- ❌ Запускать frontend как процесс в production
- ❌ Использовать Vite dev server в production
- ❌ Два процесса PM2 (frontend + backend)

---

## ✅ ИТОГО:

- **Backend:** один процесс PM2, работает постоянно
- **Frontend:** статические файлы в `dist/`, раздаются через Nginx
- **Сборка frontend:** делается один раз при деплое

**PM2 = только backend!**
