# Развертывание и эксплуатация Poizonic на Timeweb (для ВКР)

## 1. Общая схема размещения

Проект развёрнут по классической схеме "клиент + backend API + БД":

- frontend (Telegram Mini App, React/Vite) собирается в статические файлы;
- backend (Node.js + Express) работает как отдельный сервис;
- reverse proxy и HTTPS выполняет Nginx;
- процессы backend управляются PM2 (с авто-перезапуском);
- DNS-домен указывает на сервер;
- SSL-сертификаты выпущены через Certbot (Let's Encrypt);
- база данных размещена отдельно как арендуемый сервис Timeweb (отдельный экземпляр от VPS).

Итоговая инфраструктура полностью размещена в контуре Timeweb:

1) VPS-сервер (приложение: Nginx + PM2 + Node.js),
2) отдельная managed MySQL-база,
3) внешний домен с TLS.

## 2. Backend как сервис (PM2)

Backend запускается не вручную в терминале, а как фоновый сервис через PM2.  
Это обеспечивает стабильность и эксплуатационную управляемость:

- автоматический перезапуск при сбое процесса;
- возможность автозапуска после перезагрузки ОС;
- централизованные логи и мониторинг процесса;
- безостановочный операционный режим (service-like модель).

Практически это означает, что API постоянно доступен по внутреннему порту (например, 3000), а наружу публикуется через Nginx.

## 3. Frontend: отдельная сборка

Frontend разворачивается как результат production-сборки:

- команда сборки: `npm run build`;
- артефакты сборки размещаются в каталоге статики;
- Nginx отдает эти файлы пользователю.

Важно: frontend и backend обновляются независимо:

- backend можно перезапустить через PM2;
- frontend обновляется заменой статической сборки.

Такая модель проста в сопровождении и хорошо подходит для ВКР-проекта.

## 4. Nginx в архитектуре

Nginx выполняет несколько ключевых функций:

- принимает входящий HTTP/HTTPS-трафик;
- обслуживает статические файлы frontend;
- проксирует запросы `/api/*` на backend (Node.js/Express);
- поддерживает SSL-конфигурацию и безопасные TLS-настройки;
- участвует в механизме эксплуатационной надежности (вместе с PM2).

Фактически Nginx является единой точкой входа в приложение.

## 5. SSL/TLS и домен

Используется собственный домен, на котором доступно приложение.  
Сертификаты TLS выпущены через **Certbot** (Let's Encrypt), что обеспечивает:

- шифрование канала (HTTPS);
- корректную работу Telegram Mini App (требование безопасного контекста);
- доверенную публикацию API и клиентской части.

В ВКР корректная формулировка:  
"SSL-сертификаты выпущены и обслуживаются с использованием Certbot (Let's Encrypt)".

## 6. База данных (отдельный сервис Timeweb)

БД развернута отдельно от VPS как арендуемый сервис Timeweb (managed MySQL).  
Это даёт следующие преимущества:

- логическое разделение приложения и данных;
- независимое администрирование БД;
- упрощённое резервирование и сопровождение на стороне провайдера;
- снижение рисков при перезапуске/обслуживании VPS-приложения.

Backend подключается к этой БД по сетевым параметрам доступа (host, user, password, db name).

## 7. Доставка кода на сервер

Код публикуется на сервер через GitHub (модель Git-based deploy):

1. изменения коммитятся и отправляются в удалённый репозиторий;
2. на сервере забирается актуальная версия проекта;
3. выполняются сборка frontend и перезапуск backend-процесса.

Таким образом, GitHub выступает источником версий и каналом доставки релизов.

## 8. Автоперезапуск и отказоустойчивость

Механизм автоперезапуска реализован на уровне PM2.  
Даже при аварийном завершении backend-процесса сервис автоматически поднимается повторно.

В связке с Nginx это формирует базовый уровень эксплуатационной устойчивости:

- backend управляется менеджером процессов;
- web-трафик стабильно обслуживается reverse proxy;
- обновления выполняются контролируемо.

## 9. Проверка твоего исходного описания

Твоё описание в целом корректное. Ниже только правки формулировок:

- правильно: `npm run build` (а не "npm ran build");
- правильно: `Certbot` (а не "cerbota");
- корректно указывать, что backend работает как сервис через PM2, frontend собирается отдельно, Nginx настроен как reverse proxy, SSL включён, деплой идёт через GitHub, инфраструктура и БД размещены в Timeweb (БД отдельно от VPS).

## 10. Готовая формулировка для вставки в ВКР

"Информационная система развернута на VPS-площадке Timeweb. Серверная часть (Node.js/Express) работает как фоновый сервис под управлением PM2 с автоматическим перезапуском при сбоях и после перезагрузки сервера. Клиентская часть (Telegram Mini App на React/Vite) собирается отдельной командой `npm run build` и публикуется как статический фронтенд. Входящий трафик обрабатывается Nginx, который выполняет функции reverse proxy для API и web-сервера для статических файлов. Для домена настроен HTTPS; SSL-сертификаты получены через Certbot (Let's Encrypt). База данных MySQL размещена как отдельный арендуемый сервис Timeweb и подключается к backend по сетевым параметрам. Поставка обновлений выполняется через GitHub-репозиторий."
