## 2. Настройка домена

### 2.1. Настройка DNS на Reg.ru

1. Зайдите в панель управления Reg.ru
2. Перейдите в раздел **DNS-серверы**
3. Добавьте/измените A-запись:
   - **Тип**: A
   - **Поддомен**: @ (или ваш поддомен, например: `app`)
   - **IP-адрес**: IP вашего сервера Timeweb
   - **TTL**: 3600

4. Если используете поддомен (например, `app.вашдомен.ru`):
   - **Тип**: A
   - **Поддомен**: `app`
   - **IP-адрес**: IP вашего сервера
   - **TTL**: 3600

5. Сохраните изменения и подождите 5-30 минут для распространения DNS

### 2.2. Проверка DNS

Проверьте, что домен указывает на ваш IP:

```bash
nslookup вашдомен.ru
# или
dig вашдомен.ru
```

## 3. Установка необходимого ПО

### 3.1. Установка Node.js (версия 18+)

```bash
# Установка Node.js через NodeSource
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# Проверка версии
node -v  # Должно быть v18.x.x или выше
npm -v
```

### 3.2. Установка MySQL

```bash
# Установка MySQL
sudo apt install -y mysql-server

# Запуск и автозапуск MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# Настройка безопасности MySQL
sudo mysql_secure_installation
```

**Важно**: Во время настройки безопасности:
- Установите надежный пароль для root
- Выберите все рекомендуемые опции безопасности

### 3.3. Установка PM2 (менеджер процессов Node.js)

```bash
sudo npm install -g pm2

# Настройка автозапуска PM2 при перезагрузке сервера
pm2 startup
# Скопируйте и выполните команду, которую выведет PM2
```

### 3.4. Установка Nginx

```bash
sudo apt install -y nginx

# Запуск и автозапуск Nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# Проверка статуса
sudo systemctl status nginx
```

### 3.5. Установка Certbot (для SSL сертификатов)

```bash
# Установка Certbot
```

## 5. Настройка базы данных

> **Примечание**: Если у вас уже есть созданная и частично заполненная база данных, пропустите шаги 5.1 и 5.2. Перейдите сразу к проверке подключения (5.3).

### 5.1. Создание базы данных и пользователя (если БД еще нет)

```bash
# Войдите в MySQL
sudo mysql -u root -p
```

В консоли MySQL выполните:

```sql
-- Создание базы данных
CREATE DATABASE default_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Создание пользователя (замените на свои данные)
CREATE USER 'gen_user'@'localhost' IDENTIFIED BY 'ваш_надежный_пароль';
CREATE USER 'gen_user'@'%' IDENTIFIED BY 'ваш_надежный_пароль';

-- Выдача прав пользователю
GRANT ALL PRIVILEGES ON default_db.* TO 'gen_user'@'localhost';
GRANT ALL PRIVILEGES ON default_db.* TO 'gen_user'@'%';

-- Применение изменений
FLUSH PRIVILEGES;

-- Выход
EXIT;
```

### 5.2. Импорт схемы базы данных (только если БД пустая)

**Если база данных пустая и нужно создать структуру:**

```bash
# Вернитесь в директорию проекта
cd /var/www/poizonic-mini-app/poizonic-mini-app_reserve_copy

# Импортируйте схему базы данных (только если БД пустая!)
mysql -u gen_user -p default_db < database/schema.sql
```

**Если у вас уже есть заполненная БД:**
- **Пропустите этот шаг**, чтобы не потерять данные
- Убедитесь, что структура таблиц соответствует текущей версии приложения

### 5.3. Проверка подключения к БД

```bash
mysql -u gen_user -p -h localhost default_db

# В MySQL консоли проверьте таблицы
SHOW TABLES;
EXIT;
```

## 9. Настройка Nginx

### 9.1. Создание конфигурации Nginx

```bash
# Создайте конфигурационный файл для вашего домена
sudo nano /etc/nginx/sites-available/poizonic
```

**Содержимое файла (замените `вашдомен.ru` на ваш домен):**

```nginx
# Редирект с HTTP на HTTPS
server {
    listen 80;
    server_name вашдомен.ru www.вашдомен.ru;
    
    # Для получения сертификата Let's Encrypt
    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }
    
    location / {
        return 301 https://$server_name$request_uri;
    }
}

# HTTPS конфигурация
server {
    listen 443 ssl http2;
    server_name вашдомен.ru www.вашдомен.ru;

    # SSL сертификаты (будут добавлены Certbot)
    # ssl_certificate /etc/letsencrypt/live/вашдомен.ru/fullchain.pem;
    # ssl_certificate_key /etc/letsencrypt/live/вашдомен.ru/privkey.pem;

    # Настройки безопасности SSL
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Размер загружаемых файлов
    client_max_body_size 10M;

    # Статические файлы Frontend
    location / {
        root /var/www/poizonic-mini-app/poizonic-mini-app_reserve_copy/frontend/dist;
        try_files $uri $uri/ /index.html;
        index index.html;
        
        # Кэширование статических файлов
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API запросы к Backend
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # Таймауты для долгих запросов
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Загруженные файлы (отзывы)
    location /uploads {
        alias /var/www/poizonic-mini-app/poizonic-mini-app_reserve_copy/backend/uploads;
        expires 1y;
        add_header Cache-Control "public";
    }

    # Логи
    access_log /var/log/nginx/poizonic-access.log;
    error_log /var/log/nginx/poizonic-error.log;
}
```

Сохраните файл: `Ctrl+O`, `Enter`, `Ctrl+X`

### 9.2. Активация конфигурации

```bash
# Создайте симлинк
sudo ln -s /etc/nginx/sites-available/poizonic /etc/nginx/sites-enabled/

# Удалите дефолтную конфигурацию (если нужно)
sudo rm /etc/nginx/sites-enabled/default

# Проверьте конфигурацию на ошибки
sudo nginx -t

# Если проверка прошла успешно, перезагрузите Nginx
sudo systemctl reload nginx
```

## 11. Настройка автоматического деплоя (CI/CD)

Теперь настроим автоматическое обновление приложения при пуше кода в GitHub. Есть два варианта:

### Вариант A: GitHub Actions с SSH (рекомендуется)

Этот вариант использует GitHub Actions для автоматического деплоя через SSH.

#### 11.1. Подготовка SSH ключа на сервере

```bash
# На сервере создайте SSH ключ для деплоя
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/github_actions_deploy

# Покажите публичный ключ
cat ~/.ssh/github_actions_deploy.pub

# Добавьте приватный ключ в авторизованные
cat ~/.ssh/github_actions_deploy.pub >> ~/.ssh/authorized_keys
```

#### 11.2. Настройка GitHub Secrets

1. Зайдите в ваш репозиторий на GitHub
2. Перейдите в **Settings** → **Secrets and variables** → **Actions**
3. Добавьте следующие секреты (New repository secret):

   - **`SERVER_HOST`**: IP адрес вашего сервера (например: `123.45.67.89`)
   - **`SERVER_USER`**: имя пользователя SSH (обычно `root` или ваш пользователь)
   - **`SSH_PRIVATE_KEY`**: приватный ключ SSH (содержимое `~/.ssh/github_actions_deploy` без `.pub`)
   - **`SERVER_PORT`**: порт SSH (обычно `22`, можно не добавлять, будет использован по умолчанию)

**Как получить приватный ключ:**
```bash
# На сервере
cat ~/.ssh/github_actions_deploy
# Скопируйте весь вывод, включая строки BEGIN и END
```

#### 11.3. Настройка workflow файла

Workflow файл уже создан в `.github/workflows/deploy.yml`. Убедитесь, что:

1. В файле указана правильная ветка (по умолчанию `main`, если у вас `master` - измените)
2. Путь к проекту правильный (`/var/www/poizonic-mini-app/poizonic-mini-app_reserve_copy`)

**Если нужно изменить ветку:**
Откройте `.github/workflows/deploy.yml` и измените:
```yaml
branches:
  - main  # или master
```

#### 11.4. Тестирование автоматического деплоя

После настройки secrets, просто сделайте коммит и пуш:

```bash
git add .
git commit -m "Test auto-deploy"
git push origin main
```

Проверьте статус деплоя:
1. Зайдите на GitHub → **Actions** вкладка
2. Увидите запущенный workflow "Deploy to Production"
3. Кликните на него, чтобы увидеть логи

### Вариант B: Webhook сервер (альтернативный)

Если вы предпочитаете использовать webhook вместо SSH ключей.

#### 11.5. Установка webhook сервера на сервер

```bash
# Перейдите в директорию проекта
cd /var/www/poizonic-mini-app/poizonic-mini-app_reserve_copy

# Скопируйте webhook сервер
cp server-deploy-webhook.js ~/

# Установите зависимости (если нужно)
cd ~
npm init -y
npm install express

# Отредактируйте конфигурацию
nano server-deploy-webhook.js
```

Измените в файле:
- `PROJECT_PATH` - путь к вашему проекту
- `DEPLOY_WEBHOOK_SECRET` - секретный ключ (сгенерируйте: `openssl rand -hex 32`)
- `DEPLOY_WEBHOOK_PORT` - порт для webhook (по умолчанию 9000)

#### 11.6. Запуск webhook сервера через PM2

```bash
# Создайте конфигурацию PM2 для webhook
pm2 start ~/server-deploy-webhook.js --name deploy-webhook
pm2 save
```

#### 11.7. Настройка Nginx для webhook

Добавьте в конфигурацию Nginx (перед `location /api`):

```nginx
# Webhook для автоматического деплоя
location /deploy {
    proxy_pass http://localhost:9000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

Перезагрузите Nginx:
```bash
sudo nginx -t
sudo systemctl reload nginx
```

#### 11.8. Настройка GitHub Webhook

1. Зайдите в репозиторий → **Settings** → **Webhooks** → **Add webhook**
2. **Payload URL**: `https://вашдомен.ru/deploy`
3. **Content type**: `application/json`
4. **Secret**: тот же секретный ключ, что в `server-deploy-webhook.js`
5. **Events**: выберите "Just the push event"
6. **Active**: включите галочку
7. Нажмите **Add webhook**

#### 11.9. Настройка GitHub Actions для webhook (опционально)

Используйте workflow `.github/workflows/deploy-webhook.yml`:

1. Добавьте секреты:
   - **`DEPLOY_WEBHOOK_URL`**: `https://вашдомен.ru/deploy`
   - **`DEPLOY_WEBHOOK_SECRET`**: ваш секретный ключ

2. Workflow будет вызывать webhook при пуше

### 11.10. Проверка автоматического деплоя

```bash
# На сервере смотрите логи webhook (если используете вариант B)
pm2 logs deploy-webhook

# Или логи PM2 основного приложения
pm2 logs poizonic-backend
```

**Теперь при каждом пуше в GitHub приложение будет автоматически обновляться на сервере!**

## 🔧 Решение возможных проблем

### Проблема: Сервер не запускается

```bash
# Проверьте логи PM2
pm2 logs poizonic-backend --err

# Проверьте, занят ли порт 3000
sudo lsof -i :3000

# Проверьте .env файл
cat backend/.env
```

### Проблема: Nginx возвращает 502 Bad Gateway

1. Проверьте, запущен ли Backend:
   ```bash
   pm2 status
   ```

2. Проверьте, отвечает ли Backend на localhost:
   ```bash
   curl http://localhost:3000/api/health
   ```

3. Проверьте логи Nginx:
   ```bash
   sudo tail -f /var/log/nginx/poizonic-error.log
   ```

### Проблема: SSL сертификат не работает

```bash
# Проверьте статус Certbot
sudo certbot certificates

# Обновите сертификат вручную
sudo certbot renew

# Проверьте конфигурацию Nginx
sudo nginx -t
```

### Проблема: Frontend не загружается

1. Проверьте, собран ли Frontend:
   ```bash
   ls -la frontend/dist/
   ```

2. Проверьте права доступа:
   ```bash
   sudo chown -R www-data:www-data frontend/dist/
   ```

3. Проверьте конфигурацию Nginx на правильный путь

### Проблема: База данных недоступна

```bash
# Проверьте статус MySQL
sudo systemctl status mysql

# Проверьте подключение
mysql -u gen_user -p -h localhost default_db

# Проверьте логи MySQL
sudo tail -f /var/log/mysql/error.log
```
