# Исправление ошибки 413 при загрузке изображений

## Проблема
При загрузке 8+ изображений возникает ошибка 413 "Payload Too Large". Это происходит потому, что Nginx блокирует большие запросы до того, как они попадут в приложение Node.js.

## Решение

### 1. Найдите конфигурационный файл Nginx

Обычно это один из файлов:
- `/etc/nginx/sites-available/default`
- `/etc/nginx/sites-enabled/default`
- `/etc/nginx/nginx.conf`
- `/etc/nginx/conf.d/default.conf`

### 2. Найдите блок `server {}` для вашего приложения

Ищите блок, который содержит `proxy_pass` на ваш Node.js сервер (обычно `localhost:3000`).

### 3. Добавьте или измените `client_max_body_size`

Внутри блока `server {}` добавьте (или измените существующую строку):

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # Увеличиваем лимит загрузки файлов до 200MB
    client_max_body_size 200M;
    
    # Также увеличьте таймауты для больших запросов
    proxy_read_timeout 300s;
    proxy_connect_timeout 300s;
    proxy_send_timeout 300s;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4. Проверьте конфигурацию и перезагрузите Nginx

```bash
# Проверка синтаксиса
nginx -t

# Если всё ОК, перезагрузите Nginx
systemctl reload nginx

# Или перезапустите
systemctl restart nginx
```

### 5. Альтернативный способ (если используете скрипт)

Выполните на сервере:
```bash
cd /var/www/poizonic-mini-app
bash fix-nginx-upload-limit.sh
```

## Проверка

После исправления Nginx попробуйте снова загрузить 8 изображений. В логах сервера вы увидите:
- Размер запроса
- Количество файлов
- Общий размер всех файлов
- Размер каждого файла

Это поможет диагностировать проблемы в будущем.
