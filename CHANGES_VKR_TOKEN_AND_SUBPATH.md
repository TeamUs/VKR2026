# Изменения для ВКР: токен, подпуть /vkr и сервер

Краткий чек-лист того, что нужно поменять, чтобы приложение работало под путём **/vkr** на одном домене и с **новым Telegram-токеном**.

---

## ⚠️ Безопасность токена

- **Не коммитить** токен в репозиторий. Хранить только в **.env** в папке backend (локально и на сервере).
- В **backend/.env** указать: `BOT_TOKEN=ваш_новый_токен` (тот, что выдали в BotFather для бота ВКР).
- Если токен мог попасть в чат или в git — в BotFather: **Revoke current token** и создать новый.

---

## 1. Telegram (BotFather)

- Создан бот для ВКР, получен новый токен (уже есть).
- В настройках бота указать **Mini App URL**:  
  `https://ВАШ_ДОМЕН/vkr/`  
  (обязательно со слэшем в конце).
- Токен прописать только в **.env** бэкенда (см. ниже).

---

## 2. Frontend (подпуть /vkr и обращение к API)

Чтобы приложение открывалось по адресу `https://домен/vkr/`, а запросы шли на `https://домен/vkr/api/...`:

| Что | Где | Что сделать |
|-----|-----|-------------|
| Базовый путь сборки | `frontend/vite.config.ts` | Используется `base: process.env.VITE_APP_BASE_PATH \|\| './'`. При сборке под /vkr задаётся через `.env.production`. |
| Вызовы API | Компоненты и App | Использованы **относительные** пути: `fetch('api/...')` вместо `fetch('/api/...')`. Тогда при открытии с `https://домен/vkr/` запрос `api/user/init` уходит на `https://домен/vkr/api/user/init`. Никакого отдельного apiClient не нужно. |
| Переменная для сборки | `frontend/.env.production` | Уже добавлена: `VITE_APP_BASE_PATH=/vkr/`. При `npm run build` подпуть подхватится автоматически. |

В разработке (`npm run dev`) страница на `http://localhost:3001/`, запрос `api/...` идёт на `http://localhost:3001/api/...`, прокси переправляет на бэкенд — всё работает без доп. настроек.

---

## 3. Backend (Node)

Бэкенд **не** вешает роуты на `/vkr` — он по-прежнему отдаёт только `/api/...`. Подпуть `/vkr/api` делает nginx (проксирует на бэкенд).

| Что | Где | Что сделать |
|-----|-----|-------------|
| Токен бота | `.env` на сервере и локально для ВКР | `BOT_TOKEN=новый_токен_от_BotFather`. |
| CORS | `server.js` (cors origin) | Добавить в список origin адрес с /vkr, например: `https://ВАШ_ДОМЕН` (достаточно домена, путь браузер не шлёт в Origin). Либо задать через `CORS_ORIGINS` в .env. |
| FRONTEND_URL | `.env` на сервере | Указать полный URL приложения, например: `https://ВАШ_ДОМЕН/vkr/` (если бэкенд где-то формирует ссылки на фронт). |
| Порт | `.env` на сервере | Для ВКР использовать отдельный порт, например `PORT=3002` (чтобы не пересекаться с основным приложением на 3000). |

Итог: в коде бэкенда менять пути API не нужно — только конфиг (.env и при необходимости CORS).

---

## 4. Сервер (Nginx)

Один домен, два приложения: основное (например, в корне или под своим путём) и ВКР под **/vkr**.

- **Статика фронта ВКР** — отдавать папку со сборкой (например, `.../vkr-frontend/dist`) по префиксу `/vkr/`:

```nginx
location /vkr/ {
    alias /путь/к/vkr-frontend/dist/;   # слэш в конце важен
    try_files $uri $uri/ /vkr/index.html;
}
```

- **API ВКР** — проксировать запросы с `/vkr/api/` на бэкенд ВКР (порт, например, 3002):

```nginx
location /vkr/api/ {
    proxy_pass http://127.0.0.1:3002/api/;   # слэш в конце — nginx подставит путь
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

В результате запрос `https://домен/vkr/api/user/init` уходит на бэкенд как `http://127.0.0.1:3002/api/user/init`. Роуты в Express менять не нужно.

---

## 5. Сборка и деплой

1. **Сборка фронта для /vkr:**

   В папке `frontend` создать `.env.production` (или задать при сборке):

   ```
   VITE_API_BASE=/vkr
   ```

   Затем:

   ```bash
   cd frontend && npm run build
   ```

   В `vite.config.ts` должно быть `base: '/vkr/'` — тогда все ресурсы (JS/CSS) будут с путём `/vkr/assets/...`.

2. **Бэкенд на сервере:**  
   Клонировать репозиторий VKR2026, в корне бэкенда создать `.env` с `BOT_TOKEN`, `PORT=3002`, `DB_*`, `FRONTEND_URL`, `CORS_ORIGINS` при необходимости. Запуск как у основного приложения (node/server.js или PM2).

3. **Nginx:**  
   Настроить `location /vkr/` и `location /vkr/api/` как выше, перезагрузить nginx.

4. **BotFather:**  
   Mini App URL = `https://ВАШ_ДОМЕН/vkr/`.

## 6. Краткий чек-лист

- [ ] Новый токен записан только в `.env`, не в коде и не в git.
- [ ] В BotFather: Mini App URL = `https://ДОМЕН/vkr/`.
- [ ] Frontend: `base: '/vkr/'`, все вызовы API через `apiFetch`, при сборке задан `VITE_API_BASE=/vkr`.
- [ ] Backend: в `.env` на сервере — новый `BOT_TOKEN`, свой `PORT`, `FRONTEND_URL` с /vkr, при необходимости CORS.
- [ ] Nginx: раздаёт статику по `/vkr/`, проксирует `/vkr/api/` на порт бэкенда ВКР.
- [ ] Открыть в браузере `https://ДОМЕН/vkr/` — загружается приложение, запросы идут на `/vkr/api/...`.

После этого можно спокойно вносить изменения под диплом (удаление раздела «Покупка юаней», ИИ-агент и т.д.).
