## 2. ❌ requirements.txt (Python файл в Node.js проекте!)

### Проблема:
Файл `requirements.txt` содержит зависимости для **Python** проекта (aiogram, selenium и т.д.), но этот проект - **Node.js/React**.

### Анализ содержимого:
```
- aiogram==3.13.1 (Python Telegram bot)
- selenium==4.25.0 (Python WebDriver)
- mysql-connector-python==9.1.0 (Python MySQL)
- И другие Python пакеты...
```

### Возможные проблемы:
1. **Если на сервере запускается `pip install -r requirements.txt`** - это устанавливает ненужные Python пакеты
2. **requirements.txt может быть из другого проекта** - случайно попал в репозиторий
3. **Не влияет напрямую на стили**, но может создавать путаницу

### Решение:
1. Убедиться, что на сервере **НЕ запускается** `pip install`
2. Использовать только `npm install` для Node.js зависимостей
3. Переместить или удалить `requirements.txt` (если он не нужен)

### Проверка на сервере:
```bash
# Проверить, не установлены ли Python пакеты
pip list | grep -i "aiogram\|selenium"

# Проверить, что используется только npm
cd /var/www/poizonic-mini-app/frontend
npm list styled-components
```

## 3. 🖥️ Windows vs Ubuntu - РАЗЛИЧИЯ В СБОРКЕ

### ⚠️ Это самая вероятная причина проблем!

### 3.1. Различия в путях к файлам

#### Проблема: Файл с пробелом в имени
```
/Music Warrior.ttf
```

На Windows: работает
На Linux: **пробелы в именах файлов могут вызывать проблемы!**

#### Проверка на сервере:
```bash
cd /var/www/poizonic-mini-app/frontend/public
ls -la | grep -i "music\|warrior"

# Проверить, существует ли файл
ls -la "Music Warrior.ttf" 2>/dev/null || echo "Файл не найден!"

# Проверить в dist
ls -la /var/www/poizonic-mini-app/frontend/dist/ | grep -i "music\|warrior"
```

### 3.2. Различия в package-lock.json

#### Проблема:
- **Windows**: package-lock.json создан на Windows
- **Ubuntu**: npm может устанавливать другие версии зависимостей

#### Решение:
```bash
cd /var/www/poizonic-mini-app/frontend
rm -rf node_modules package-lock.json
npm ci  # Использует точные версии из package-lock.json из репозитория
```

### 3.3. Различия в правах доступа

#### Проблема:
- **Windows**: не чувствителен к правам
- **Linux**: строгие права доступа

#### Проверка:
```bash
# Проверить права на файлы
ls -la /var/www/poizonic-mini-app/frontend/dist/
ls -la /var/www/poizonic-mini-app/frontend/node_modules/.bin/

# Исправить права (если нужно)
chmod -R 755 /var/www/poizonic-mini-app/frontend/dist
chmod -R 755 /var/www/poizonic-mini-app/frontend/node_modules/.bin
```

### 3.4. Различия в версиях Node.js

#### Проблема:
- Локально: может быть другая версия Node.js
- На сервере: другая версия Node.js

#### Проверка:
```bash
# На сервере
node -v
npm -v

# Локально (Windows)
node -v
npm -v
```

**Должны совпадать или быть близкими версиями!**

### 3.5. Кеш Vite и различия в сборке

#### Проблема:
- Vite кеширует сборку на Windows
- На Linux может быть другой кеш

#### Решение:
```bash
cd /var/www/poizonic-mini-app/frontend
rm -rf dist .vite node_modules/.vite
npm run build
```

### 3.6. Различия в обработке стилей

#### Проблема:
- Windows: путь может быть `C:\Users\...`
- Linux: путь может быть `/var/www/...`
- **styled-components может не правильно инициализироваться**

#### Решение:
Убедиться, что:
1. `index.tsx` создает style элемент до рендера
2. `GlobalStyle` рендерится правильно
3. Нет проблем с путями к шрифтам


### Шаг 4: Полная пересборка
```bash
cd /var/www/poizonic-mini-app/frontend

# 1. Удалить все кеши
rm -rf node_modules package-lock.json dist .vite node_modules/.vite

# 2. Получить свежий package-lock.json из Git
git checkout frontend/package-lock.json

# 3. Установить зависимости
npm ci

# 4. Проверить версию styled-components
npm list styled-components
# Должно быть: styled-components@6.1.13

# 5. Собрать проект
npm run build

# 6. Проверить результат
ls -la dist/
ls -la dist/assets/
```

### Шаг 5: Проверить права доступа
```bash
# На сервере
sudo chown -R www-data:www-data /var/www/poizonic-mini-app/frontend/dist
chmod -R 755 /var/www/poizonic-mini-app/frontend/dist

# Проверить, что nginx может читать
sudo -u www-data ls -la /var/www/poizonic-mini-app/frontend/dist/
```

### Шаг 6: Перезагрузить Nginx
```bash
sudo systemctl reload nginx
```

## 🔧 ДОПОЛНИТЕЛЬНЫЕ ИСПРАВЛЕНИЯ

### Исправить путь к шрифту (без пробела)

Если файл `Music Warrior.ttf` вызывает проблемы:

1. Переименовать файл (рекомендуется):
```bash
# На сервере
cd /var/www/poizonic-mini-app/frontend/public
mv "Music Warrior.ttf" "MusicWarrior.ttf"
```

2. Обновить `fonts.css`:
```css
@font-face {
  font-family: 'Music Warrior';
  src: url('/MusicWarrior.ttf') format('truetype');
  font-weight: normal;
  font-style: normal;
  font-display: swap;
}
```

3. Обновить в `dist`:
```bash
cd /var/www/poizonic-mini-app/frontend/dist
mv "Music Warrior.ttf" "MusicWarrior.ttf" 2>/dev/null
```

### Создать скрипт для проверки различий

Создать `check-differences.sh`:
```bash
#!/bin/bash
echo "=== Проверка различий Windows/Ubuntu ==="
echo ""
echo "Node.js версия: $(node -v)"
echo "npm версия: $(npm -v)"
echo ""
echo "=== Проверка файлов ==="
cd /var/www/poizonic-mini-app/frontend
echo "Music Warrior.ttf в public:"
ls -la public/ | grep -i music || echo "НЕ НАЙДЕН"
echo ""
echo "Music Warrior.ttf в dist:"
ls -la dist/ | grep -i music || echo "НЕ НАЙДЕН"
echo ""
echo "=== Проверка зависимостей ==="
npm list styled-components
echo ""
echo "=== Проверка прав доступа ==="
ls -ld dist/
ls -ld node_modules/.bin/
```
