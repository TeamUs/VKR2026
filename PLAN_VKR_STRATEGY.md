# План-стратегия: мини-приложение для ВКР (VKR2026)

Репозиторий: https://github.com/TeamUs/VKR2026  
Исходный проект: `mini_app_vkr` (копия основного приложения без раздела «Покупка юаней»).

---

## Цели

1. Запушить проект в GitHub (VKR2026).
2. Сохранить всю структуру приложения **кроме** раздела «Покупка юаней».
3. Полностью убрать раздел «Покупка юаней» (фронт, бэкенд, навигация, статистика, админка).
4. Подготовить базу для внедрения ИИ-агента на базе знаний специальности.
5. Использовать отдельный Telegram-токен для мини-приложения ВКР.
6. Организовать отдельные сервисы на сервере для стабильной работы.

---

## Фаза 1: Git и первый пуш в GitHub

### 1.1 Настроить remote для VKR2026

Сейчас в проекте `mini_app_vkr` настроен origin на `poizonic-mini-app`. Для диплома нужен отдельный репозиторий.

**Вариант A (рекомендуется):** добавить второй remote и пушить в него:

```bash
cd "C:\Users\Артём\Desktop\Учеба\ВКР\Проект\mini_app_vkr"
git remote add vkr https://github.com/TeamUs/VKR2026.git
git push -u vkr main
```

Дальше для ВКР можно пушить: `git push vkr main`.

**Вариант B:** если эта копия только для ВКР и с poizonic больше не синхронизируете:

```bash
git remote set-url origin https://github.com/TeamUs/VKR2026.git
git push -u origin main
```

### 1.2 Перед первым пушем

- Убедиться, что в репозиторий **не** попадают:
  - `.env` (уже в `.gitignore`),
  - дампы БД с реальными данными (при необходимости добавить в `.gitignore` строки вида `dump-*.sql` или папку `database/dumps/`).
- Добавить в корень **README.md** с кратким описанием проекта ВКР и инструкцией по запуску (опционально, но полезно для диплома и проверки).
- При необходимости создать файл **.env.example** с перечнем переменных (без значений секретов), чтобы любой мог поднять копию.

**Итог фазы 1:** код лежит в https://github.com/TeamUs/VKR2026, структура проекта сохранена.

### 2.3 База данных

- **Вариант A (минимальные изменения):** таблицу `yuan_purchases` в продакшене ВКР не использовать: не создавать её при деплое или оставить пустой. В коде к ней не обращаться. В `schema.sql` можно оставить определение с комментарием «Не используется в версии ВКР» или удалить.
- **Вариант B (чистая схема для диплома):** создать миграцию или отдельный `schema_vkr.sql` без таблицы `yuan_purchases` и без полей, которые использовались только для юаней (если такие есть в других таблицах). Для новых инсталляций использовать эту схему.

Рекомендация: на этапе диплома достаточно не вызывать API/таблицу юаней (вариант A); при защите можно показать схему без финансового раздела.

### 2.4 Чек-лист после удаления

- [ ] В приложении нет вкладки/кнопок «Купить юань» и «Курс юаня».
- [ ] Профиль открывается без ошибок, нет обращений к `/api/yuan-purchases`, статистика без юаней.
- [ ] Админ-панель: нет вкладки и фильтров по покупкам юаней, подтверждение/отмена только для заказов.
- [ ] Бэкенд не отдаёт эндпоинты курса и покупки юаней; профиль и геймификация не используют yuan-поля.

## Фаза 4: Новый Telegram-токен для мини-приложения ВКР

- В [@BotFather](https://t.me/BotFather) создать нового бота для ВКР (или использовать существующий «учебный» бот).
- Получить токен и прописать его в `.env` проекта `mini_app_vkr` (например, `TELEGRAM_BOT_TOKEN=...`).
- В настройках бота в BotFather привязать мини-приложение (Mini App URL — ваш фронт ВКР).
- В коде использовать только этот токен для верификации Telegram Web App (initData) и при необходимости для отправки уведомлений. Не смешивать с токеном основного бота poizonic.

## Фаза 6: Сервер и сервисы

- **Один сервер для ВКР:** разнести сервисы так, чтобы основной продакшен (poizonic) не зависел от дипломного.
- Рекомендуемая схема:
  - **Backend ВКР** — отдельный процесс/контейнер (Node), свой порт, своя БД (или отдельная схема/база на том же MySQL).
  - **Frontend ВКР** — статика (Nginx или отдача из backend), свой домен/поддомен или путь (например, `vkr.ваш-домен.ru`).
  - **ИИ-агент** — отдельный сервис (контейнер/процесс), к которому backend ВКР ходит по HTTP; при необходимости очередь запросов и лимиты.
- Прокси (Nginx): маршрутизация по домену/пути на backend ВКР и на ИИ-сервис.
- Процесс-менеджер (systemd, PM2 или Docker Compose): автозапуск и перезапуск backend ВКР и ИИ-сервиса.
- Мониторинг: логи, проверка доступности эндпоинтов (health check), при необходимости алерты.

- Делать удаление юаней в отдельных коммитах (например: «frontend: remove yuan tab and exchange rate», «backend: remove yuan purchase API», «profile: remove yuan stats and achievements»), чтобы при необходимости откатить только часть изменений.
