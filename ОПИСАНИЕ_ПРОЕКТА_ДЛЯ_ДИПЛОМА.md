# ОПИСАНИЕ ПРОЕКТА: TELEGRAM MINI APP "POIZONIC"
## Полное техническое описание для дипломной работы

---

## 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

### 1.1. Название проекта
**Poizonic Mini App** - Telegram Mini Application для заказа товаров с китайского маркетплейса Poizon (得物)

### 1.2. Цель проекта
Разработка полнофункционального веб-приложения в формате Telegram Mini App, которое позволяет пользователям:
- Рассчитывать стоимость товаров с учетом доставки и комиссий
- Оформлять заказы товаров с китайского маркетплейса Poizon
- Отслеживать статус доставки заказов
- Покупать китайскую валюту (юань) по выгодному курсу
- Участвовать в реферальной программе
- Получать достижения и повышать уровень через систему геймификации
- Просматривать отзывы других пользователей и оставлять свои

### 1.3. Актуальность проекта
- Рост популярности китайских маркетплейсов (Poizon, Dewu) среди российских покупателей
- Необходимость упрощения процесса заказа и расчета стоимости
- Интеграция с Telegram как наиболее популярным мессенджером в России
- Использование геймификации для повышения вовлеченности пользователей

### 1.4. Целевая аудитория
- Пользователи Telegram, интересующиеся покупками товаров из Китая
- Любители брендовой одежды, обуви и аксессуаров
- Пользователи, желающие сэкономить на покупках через выгодный курс валют

## 2. ТЕХНОЛОГИЧЕСКИЙ СТЕК

### 2.1. Backend (Серверная часть)

**Основные технологии:**
- **Node.js** (версия 18+) - серверная платформа
- **Express.js** (v4.18.2) - веб-фреймворк для создания REST API
- **MySQL2** (v3.6.5) - драйвер для работы с базой данных MySQL
- **Multer** (v1.4.5) - middleware для загрузки файлов (отзывы с фото/видео)
- **Axios** (v1.12.2) - HTTP клиент для внешних API запросов
- **Node-cron** (v4.2.1) - планировщик задач для автоматических операций
- **dotenv** (v16.3.1) - управление переменными окружения

**Архитектурные решения:**
- RESTful API архитектура
- Модульная структура кода (отдельные модули для геймификации, планировщика)
- Connection pooling для оптимизации работы с БД
- Транзакции MySQL для обеспечения целостности данных

### 2.2. Frontend (Клиентская часть)

**Основные технологии:**
- **React** (v18.2.0) - библиотека для создания пользовательского интерфейса
- **TypeScript** (v5.2.0) - типизированный JavaScript для повышения надежности кода
- **Vite** (v5.0.0) - современный сборщик и dev-сервер (замена Create React App)
- **Styled Components** (v6.1.13) - CSS-in-JS библиотека для стилизации компонентов
- **React Confetti** (v6.4.0) - библиотека для анимации конфетти при достижениях

**Особенности реализации:**
- Компонентный подход (17 React компонентов)
- TypeScript для типобезопасности
- Адаптивный дизайн с поддержкой темной/светлой темы
- Оптимизация сборки через Vite (tree-shaking, code splitting)

### 2.3. База данных

**СУБД:** MySQL (с кодировкой UTF8MB4 для поддержки эмодзи и китайских символов)

**Основные таблицы:**
- `users` - пользователи системы
- `orders` - заказы пользователей
- `order_items` - товары в заказах
- `delivery_tracking` - отслеживание доставки
- `exchange_rates` - курсы валют
- `yuan_purchases` - покупки юаней
- `referrals` - реферальная система
- `reviews` - отзывы пользователей
- `achievements` - достижения системы
- `user_achievements` - достижения пользователей
- `xp_history` - история начисления опыта
- `level_history` - история повышения уровней
- `discounts` - временные скидки для пользователей

### 2.4. Интеграции

**Telegram Bot API:**
- Авторизация через Telegram WebApp API
- Отправка уведомлений пользователям
- Haptic Feedback для тактильной обратной связи
- Back Button для навигации

**Внешние API:**
- API Центрального Банка России для получения курса валют
- Telegram Bot API для отправки сообщений

### 2.5. Инфраструктура и деплой

**Сервер:**
- Ubuntu Server
- Nginx - веб-сервер и reverse proxy
- PM2 - менеджер процессов Node.js для автозапуска и мониторинга
- SSL сертификаты для HTTPS

**CI/CD:**
- Git для контроля версий
- Автоматический деплой через скрипт `deploy.sh`

- Выбор категории товара (обувь, одежда, аксессуары и т.д.)

  - Курса юаня к рублю

1. Получение актуального курса юаня из БД

4. Добавление комиссии сервиса (зависит от уровня пользователя)
5. Итоговая сумма = цена в рублях + доставка + комиссия

### 4.8. Система геймификации

**Модуль:** `backend/gamification.js`

**Уровни пользователей:**
1. **Bronze** (0-999 XP) - Первый заказ без комиссии за оформление
2. **Silver** (1000-4999 XP) - Комиссия 900₽ навсегда
3. **Gold** (5000-24999 XP) - Комиссия 700₽ навсегда
4. **Platinum** (25000-99999 XP) - Комиссия 400₽ навсегда
5. **Diamond** (100000+ XP) - Комиссия 0₽ + специальные предложения

**Начисление XP:**
- Завершенный заказ: 100 XP
- Покупка юаней: 1 XP за каждые 100₽ потраченных
- Регистрация реферала: 50 XP
- Разблокировка достижения: переменное количество (зависит от достижения)
- Ежедневный вход: бонус за streak

**Достижения (21 достижение):**
- **Заказы:** Дракон-новичок, Цепь заказов, Император покупок
- **Рефералы:** Золотая цепочка, Реферальный фестиваль, Дракон-призыватель
- **Экономия:** Удачный расчет, Мастер баланса, Эконом-дракон
- **Активность:** Ежедневный ритуал, Лунный цикл, Год дракона
- **Творческие:** Ночной охотник, Комбо-дракон, Счастливый номер
- **Социальные:** Поделись мудростью, Групповой дракон, Вирусный огонь
- **Сезонные:** Новый год удачи, Летний огонь, Акция-дракон

**Особенности:**
- Автоматическая проверка достижений при каждом действии
- История начисления XP с указанием источника
- История повышения уровней
- Уведомления о новых достижениях и повышении уровня

## 5. БАЗА ДАННЫХ

### 5.1. Схема базы данных

**Основные таблицы:**

#### `users` - Пользователи
- `telegram_id` (BIGINT, PK) - ID пользователя в Telegram
- `commission` (INT) - Комиссия пользователя (зависит от уровня)
- `access_expires_at` (TIMESTAMP) - Срок действия доступа
- `referred_by` (BIGINT) - ID пользователя, который пригласил
- `xp` (INT) - Количество опыта (XP)
- `current_level` (VARCHAR) - Текущий уровень (Bronze-Silver-Gold-Platinum-Diamond)
- `last_daily_login` (DATE) - Дата последнего входа
- `login_streak` (INT) - Серия ежедневных входов

## 6. API ENDPOINTS

### 6.1. Аутентификация и авторизация

Все запросы (кроме публичных) требуют передачи `x-telegram-init-data` в заголовках, который содержит данные пользователя из Telegram WebApp API.

**Механизм:**
1. Telegram WebApp предоставляет `initData` при загрузке приложения
2. Frontend передает `initData` в заголовке `x-telegram-init-data`
3. Backend парсит `initData` и извлекает `telegram_id` пользователя
4. Проверяется принадлежность данных пользователю

### 6.2. Основные API Endpoints

#### Пользователи
- `GET /api/user/profile` - Получение профиля пользователя
- `GET /api/user/orders` - Список заказов пользователя
- `GET /api/user/gamification` - Данные геймификации
- `GET /api/user/referrals` - Статистика по рефералам
- `GET /api/user/heartbeat` - Проверка активности (heartbeat)

#### Заказы
- `POST /api/orders` - Создание нового заказа
- `GET /api/tracking/:trackingNumber` - Статус доставки по трек-номеру
- `GET /api/orders/:orderId` - Детали заказа

#### Расчет стоимости
- `POST /api/calculate-price` - Расчет стоимости товара
  - Body: `{ price: number, category: string }`
  - Response: `{ priceInRubles, deliveryCost, commission, totalCost, exchangeRate }`

#### Покупка юаней
- `POST /api/yuan-purchases` - Создание покупки юаней
  - Body: `{ amountCny: number, amountRub: number }`
- `GET /api/yuan-purchases` - История покупок
- `GET /api/exchange-rate` - Актуальный курс валют

#### Реферальная система
- `POST /api/referral` - Обработка реферальной регистрации
  - Body: `{ telegramId: number, referredBy: number }`
- `GET /api/user/referral-stats` - Детальная статистика по рефералам

#### Отзывы
- `GET /api/reviews` - Список отзывов (с фильтрацией по рейтингу)
- `POST /api/reviews` - Создание отзыва (с загрузкой файлов)
- `GET /api/reviews/:id` - Детали отзыва

#### Геймификация
- `POST /api/gamification/award-xp` - Начисление XP (внутренний)
- `GET /api/user/achievements` - Достижения пользователя
- `POST /api/gamification/daily-login` - Обработка ежедневного входа

#### Админ-панель
- `POST /api/admin/orders/:orderId/status` - Изменение статуса заказа
- `POST /api/admin/orders/:orderId/tracking` - Добавление трек-номера
- `POST /api/admin/reviews/:reviewId/approve` - Одобрение отзыва
- `POST /api/admin/reviews/:reviewId/reject` - Отклонение отзыва
- `POST /api/admin/exchange-rate` - Обновление курса валют
- `POST /api/admin/discounts` - Создание скидки
- `GET /api/admin/analytics/orders` - Аналитика по заказам
- `GET /api/admin/analytics/sales` - Аналитика по продажам
- `GET /api/admin/analytics/referrals` - Аналитика по рефералам

### 6.3. Обработка ошибок

- Все ошибки возвращаются в формате JSON: `{ error: "Описание ошибки" }`
- HTTP статус коды:
  - 200 - Успех
  - 400 - Ошибка валидации
  - 401 - Не авторизован
  - 404 - Не найдено
  - 500 - Внутренняя ошибка сервера

## 7. ИНТЕГРАЦИЯ С TELEGRAM

### 7.1. Telegram WebApp API

**Использование:**
- `window.Telegram.WebApp.initData` - Данные пользователя для авторизации
- `window.Telegram.WebApp.initDataUnsafe.user` - Информация о пользователе
- `window.Telegram.WebApp.HapticFeedback` - Тактильная обратная связь
- `window.Telegram.WebApp.BackButton` - Кнопка "Назад"
- `window.Telegram.WebApp.MainButton` - Главная кнопка (не используется)
- `window.Telegram.WebApp.expand()` - Развертывание приложения на весь экран
- `window.Telegram.WebApp.ready()` - Готовность приложения

### 7.2. Авторизация

**Процесс:**
1. Пользователь открывает Mini App в Telegram
2. Telegram передает `initData` в приложение
3. Frontend отправляет `initData` в заголовке `x-telegram-init-data` на backend
4. Backend валидирует данные и извлекает `telegram_id`
5. Если пользователь не существует в БД, создается новая запись

**Безопасность:**
- Проверка подлинности данных через Telegram Bot API (опционально)
- Валидация формата `initData`
- Проверка принадлежности данных пользователю

### 7.3. Уведомления

**Telegram Bot API:**
- Отправка уведомлений администратору о новых заказах
- Уведомления пользователям о статусе заказа
- Уведомления о новых достижениях и повышении уровня
- Уведомления о скором истечении скидок

## 8. СИСТЕМА ГЕЙМИФИКАЦИИ

### 8.1. Уровни и награды

**5 уровней пользователей:**

1. **Bronze** (0-999 XP)
   - Награда: Первый заказ без комиссии за оформление
   - Следующий уровень: Silver (1000 XP)

2. **Silver** (1000-4999 XP)
   - Награда: Комиссия 900₽ навсегда (вместо 1000₽)
   - Следующий уровень: Gold (5000 XP)

3. **Gold** (5000-24999 XP)
   - Награда: Комиссия 700₽ навсегда
   - Следующий уровень: Platinum (25000 XP)

4. **Platinum** (25000-99999 XP)
   - Награда: Комиссия 400₽ навсегда
   - Следующий уровень: Diamond (100000 XP)

5. **Diamond** (100000+ XP)
   - Награда: Комиссия 0₽ навсегда + специальные предложения на юань + повышенные бонусы для рефералов
   - Финальный уровень

### 8.2. Начисление XP

**Правила начисления:**
- Завершенный заказ: **100 XP**
- Покупка юаней: **1 XP за каждые 100₽** потраченных
- Регистрация реферала: **50 XP**
- Разблокировка достижения: **переменное количество** (100-5000 XP в зависимости от достижения)
- Ежедневный вход: бонус за серию входов (streak)

**Источники XP:**
- Заказы (`ORDER_COMPLETE`)
- Покупки юаней (`YUAN_PER_100RUB`)
- Рефералы (`REFERRAL_REGISTRATION`)
- Достижения (`ACHIEVEMENT_UNLOCK`)

### 8.3. Достижения (21 достижение)

**Категории достижений:**

1. **Заказы** (3 достижения)
   - Дракон-новичок - Первый заказ
   - Цепь заказов - 3 заказа за неделю
   - Император покупок - 10 заказов

2. **Рефералы** (3 достижения)
   - Золотая цепочка - Первый активный реферал
   - Реферальный фестиваль - 3 реферала за месяц
   - Дракон-призыватель - Реферал 2-го уровня

3. **Экономия** (3 достижения)
   - Удачный расчет - Сэкономьте 1,000₽ за месяц
   - Мастер баланса - 5 расчетов цен без заказа
   - Эконом-дракон - Сэкономьте 5,000₽ всего

4. **Активность** (3 достижения)
   - Ежедневный ритуал - Вход 5 дней подряд
   - Лунный цикл - Активность 28 дней
   - Год дракона - 365 дней в приложении

5. **Творческие** (3 достижения)
   - Ночной охотник - Заказ после 22:00
   - Комбо-дракон - Заказ + реферал в один день
   - Счастливый номер - 8-й заказ

6. **Социальные** (3 достижения)
   - Поделись мудростью - Поделитесь гайдом
   - Групповой дракон - Создайте чат с рефералами
   - Вирусный огонь - Пост набрал 10 лайков

7. **Сезонные** (3 достижения)
   - Новый год удачи - 3 заказа в январе
   - Летний огонь - Реферал летом
   - Акция-дракон - Заказ во время промо

### 8.4. Ежедневные входы (Streak)

- Отслеживание последовательных ежедневных входов
- Бонусы за длинные серии
- Сброс streak при пропуске дня

### 8.5. Временные скидки

- Уведомления о скором истечении скидок
- Автоматическое удаление истекших скидок
