# 🔄 ШАГ 1: Настройка PM2 для автозапуска Backend

## Цель:
Настроить PM2 так, чтобы backend автоматически запускался при перезагрузке сервера и перезапускался при ошибках.

---

## ✅ ЧТО БУДЕТ СДЕЛАНО:

1. Создан `ecosystem.config.js` для управления backend через PM2
2. Настроен автозапуск PM2 при загрузке системы
3. Настроено логирование
4. Проверена работа

---

## 🚀 ИНСТРУКЦИЯ ДЛЯ СЕРВЕРА:

### Шаг 1: Скопировать ecosystem.config.js на сервер

Файл уже создан в репозитории. Обновите код:

```bash
cd /var/www/poizonic-mini-app
git pull origin main
```

### Шаг 2: Установить PM2 (если еще не установлен)

```bash
npm install -g pm2
```

### Шаг 3: Создать папку для логов

```bash
cd /var/www/poizonic-mini-app
mkdir -p logs
chmod 755 logs
```

### Шаг 4: Остановить старый процесс backend (если запущен)

```bash
# Проверить, есть ли запущенный процесс
pm2 list

# Остановить все процессы (если есть)
pm2 stop all
pm2 delete all
```

### Шаг 5: Запустить backend через PM2

```bash
cd /var/www/poizonic-mini-app
pm2 start ecosystem.config.js
```

### Шаг 6: Сохранить конфигурацию PM2 для автозапуска

```bash
pm2 save
pm2 startup
```

Выполните команду, которую предложит `pm2 startup` (обычно что-то вроде):
```bash
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root
```

### Шаг 7: Проверить статус

```bash
pm2 status
pm2 logs poizonic-backend --lines 50
```

Должен показать:
- ✅ `online` статус
- ✅ Логи работы backend
- ✅ Время работы (uptime)

### Шаг 8: Проверить автозапуск

```bash
# Перезапустить PM2
pm2 restart all

# Или проверить, что процесс работает
pm2 list
```

### Шаг 9: Проверить работу API

```bash
# Проверить health endpoint
curl http://localhost:3000/api/health

# Или через внешний URL
curl https://poizonic.ru/api/health
```

---

## 📊 ПОЛЕЗНЫЕ КОМАНДЫ PM2:

```bash
# Посмотреть статус всех процессов
pm2 status

# Посмотреть логи backend
pm2 logs poizonic-backend

# Посмотреть последние 100 строк логов
pm2 logs poizonic-backend --lines 100

# Перезапустить backend
pm2 restart poizonic-backend

# Остановить backend
pm2 stop poizonic-backend

# Запустить backend
pm2 start poizonic-backend

# Посмотреть мониторинг в реальном времени
pm2 monit

# Посмотреть информацию о процессе
pm2 info poizonic-backend

# Перезагрузить конфигурацию
pm2 reload ecosystem.config.js
```

## ⚠️ ВОЗМОЖНЫЕ ПРОБЛЕМЫ:

### Проблема: PM2 не запускается

**Решение:**
```bash
# Проверить путь к Node.js
which node
node -v

# Проверить, что ecosystem.config.js существует
ls -la /var/www/poizonic-mini-app/ecosystem.config.js

# Проверить путь к server.js
ls -la /var/www/poizonic-mini-app/backend/server.js
```

### Проблема: Backend не отвечает

**Решение:**
```bash
# Посмотреть логи ошибок
pm2 logs poizonic-backend --err

# Проверить переменные окружения
cd /var/www/poizonic-mini-app/backend
cat .env

# Проверить, что порт 3000 свободен
netstat -tulpn | grep 3000
```

### Проблема: Автозапуск не работает

**Решение:**
```bash
# Пересоздать startup скрипт
pm2 unstartup
pm2 startup
# Выполнить команду, которую предложит PM2
```
